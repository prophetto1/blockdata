## Aside

Callout boxes for secondary information. Four types: `note` (default), `tip`, `caution`, `danger`.

<Aside>This is a **note** aside (default).</Aside>

<Aside type="tip">This is a **tip** aside.</Aside>

<Aside type="caution">This is a **caution** aside.</Aside>

<Aside type="danger">This is a **danger** aside.</Aside>

Custom title:

<Aside type="tip" title="Custom title">
  Override the default title text.
</Aside>

**Markdown shorthand** — you can also write asides without importing:

```md
:::note
This is a note.
:::

:::tip[Custom title]
This tip has a custom title.
:::
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `'note'` \| `'tip'` \| `'caution'` \| `'danger'` | `'note'` | Visual style |
| `title` | `string` | Matches `type` | Custom heading |
| `icon` | `string` | Matches `type` | Built-in icon name |

## CardGrid

Arranges cards side-by-side on wider screens.

<CardGrid>
  <Card title="First" icon="pencil">Card one content.</Card>
  <Card title="Second" icon="setting">Card two content.</Card>
  <Card title="Third" icon="puzzle">Card three content.</Card>
</CardGrid>

Works with `<LinkCard>` too:

<CardGrid>
  <LinkCard title="Getting Started" href="/getting-started/" />
  <LinkCard title="This Page" href="/reference/starlight-components/" description="You are here." />
</CardGrid>

## Code

Syntax-highlighted code blocks with titles, line highlighting, and more. Useful when rendering code from external sources (files, APIs, variables).

```js title="example.js" {2}
const greeting = 'Hello';
const name = 'BlockData';   // highlighted line
console.log(`${greeting}, ${name}!`);
```

### Fenced code block features

These work in standard Markdown fenced blocks (no import needed):

| Feature | Syntax | Example |
|---------|--------|---------|
| Language | ` ```js ` | Syntax coloring |
| Title/filename | ` ```js title="app.js" ` | Shows filename tab |
| Line highlight | ` ```js {2} ` | Highlights line 2 |
| Line range | ` ```js {2-4} ` | Highlights lines 2–4 |
| Insertions | ` ```js ins={3} ` | Green line 3 |
| Deletions | ` ```js del={2} ` | Red line 2 |
| Diff | ` ```diff ` | `+` and `-` prefixed lines |
| Word wrap | ` ```js wrap ` | Wraps long lines |
| Frame type | ` ```js frame="none" ` | Removes frame border |

### Diff example

```diff
- const old = true;
+ const updated = true;
```

### Terminal frame

```bash title="Terminal"
npm run dev
```

## FileTree

Displays directory structures with file icons and collapsible sub-trees.

<FileTree>
- src/
  - content/
    - docs/
      - index.mdx
      - **getting-started.md** highlighted file
      - reference/
        - starlight-components.mdx
  - styles/
    - global.css
- astro.config.mjs
- package.json
- ...
</FileTree>

### Features

| Feature | Syntax | Effect |
|---------|--------|--------|
| Highlight | `**filename**` | Bold with accent color |
| Comment | `filename comment text` | Gray text after name |
| Placeholder | `...` or `…` | Indicates more content |
| Special chars | `` `__init__.py` `` | Backtick-escaped names |

## Steps

Wraps an ordered list to style it as a step-by-step guide.

<Steps>
1. Install dependencies

   ```bash
   npm install
   ```

2. Start the dev server

   ```bash
   npm run dev
   ```

3. Open `http://localhost:4321` in your browser.
</Steps>

No props — just wrap any ordered list.

## Tabs

Tabbed content. Each tab needs a `label`. Use `syncKey` to synchronize multiple tab groups.

<Tabs>
  <TabItem label="npm">
    ```bash
    npm install
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```bash
    pnpm install
    ```
  </TabItem>
  <TabItem label="yarn">
    ```bash
    yarn install
    ```
  </TabItem>
</Tabs>

### Synced tabs

Multiple tab groups with the same `syncKey` stay in sync:

<Tabs syncKey="pkg">
  <TabItem label="npm">
    ```bash
    npm run dev
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```bash
    pnpm dev
    ```
  </TabItem>
</Tabs>

<Tabs syncKey="pkg">
  <TabItem label="npm">
    ```bash
    npm run build
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```bash
    pnpm build
    ```
  </TabItem>
</Tabs>

### Tabs with icons

<Tabs>
  <TabItem label="Code" icon="seti:typescript">TypeScript content.</TabItem>
  <TabItem label="Config" icon="seti:json">JSON config content.</TabItem>
</Tabs>

### Props

**`<Tabs>`**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `syncKey` | `string` | — | Synchronize tab selection across groups |

**`<TabItem>`**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | required | Tab trigger text |
| `icon` | `string` | — | Built-in icon name |
