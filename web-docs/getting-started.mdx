## What is Blockdata?

Blockdata is a schema-driven data extraction and pipeline orchestration platform. Register services, define callable functions, run document parsing pipelines, and inspect results — all through a unified API backed by a pluggable service registry.

## Get started in three steps

### Step 1: Set up your environment

<AccordionGroup>
  <Accordion icon="terminal" title="Set your base URL">
    The pipeline worker is a local FastAPI service. Export the base URL so you can use it in all requests:

    ```bash
    export BLOCKDATA_API_BASE_URL=http://localhost:8000
    ```

    <Tip>
      Add this to your shell profile so it persists across sessions.
    </Tip>
  </Accordion>
  <Accordion icon="activity" title="Confirm the worker is running">
    Run a health check to verify the pipeline worker is up and services are registered:

    ```bash
    curl $BLOCKDATA_API_BASE_URL/health
    ```

    Expected response:

    ```json
    { "status": "ok", "services": 3, "functions": 12 }
    ```

    If the worker is not running, start it from the project root:

    ```bash
    cd services/pipeline-worker && uvicorn app.main:app --reload
    ```
  </Accordion>
</AccordionGroup>

### Step 2: Explore the service registry

<AccordionGroup>
  <Accordion icon="server" title="List registered services">
    Browse all services currently registered with their health status:

    ```bash
    curl $BLOCKDATA_API_BASE_URL/registry/services
    ```
  </Accordion>
  <Accordion icon="list" title="Browse available functions">
    List all enabled functions across all services, with entrypoints and parameter schemas:

    ```bash
    curl $BLOCKDATA_API_BASE_URL/registry/functions
    ```

    <Tip>
      Filter by service type using `?service_type=document` or `?service_type=pipeline-worker`.
    </Tip>
  </Accordion>
</AccordionGroup>

### Step 3: Run your first task

<Accordion icon="play" title="Call a function">
  POST to the `/run` endpoint with a `function_name` and `parameters` matching its schema:

  ```bash
  curl -X POST $BLOCKDATA_API_BASE_URL/run \
    -H "Content-Type: application/json" \
    -d '{
      "function_name": "parse_document",
      "parameters": { "url": "https://example.com/doc.pdf" }
    }'
  ```

  The response includes a `run_id` you can use to poll for results and retrieve artifacts.
</Accordion>
