## Overview

The **unified service registry** is the single source of truth for every backend service and callable function in blockdata. It backs both the admin Services panel (`/app/settings/admin/services`) and the public browse view (`/app/services`).

**6 tables, 1 view.** All in `public` schema with RLS enabled.

| Table | Purpose |
|---|---|
| `service_type_catalog` | Enum table — service categories |
| `service_registry` | Registered service instances |
| `service_functions` | Callable functions per service |
| `service_runs` | Execution history |
| `service_run_events` | Append-only log stream per run |
| `service_run_artifacts` | Storage pointers for run outputs |
| `service_functions_view` | Joined convenience view for UI |

---

## 1. `service_type_catalog`

Enum table for service categories. Read-only for authenticated users.

```sql
CREATE TABLE public.service_type_catalog (
  service_type    TEXT PRIMARY KEY,
  label           TEXT NOT NULL,
  description     TEXT,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**Seeded values:**

| `service_type` | `label` | `description` |
|---|---|---|
| `dlt` | dlt (Load) | Data load tool — extract and load from sources to destinations |
| `dbt` | dbt (Transform) | Data build tool — SQL-based transformations on loaded data |
| `docling` | Docling (Parse) | Document parsing and conversion service |
| `edge` | Edge Functions | Supabase Edge Functions (Deno/TypeScript) |
| `conversion` | Conversion | File format conversion services |
| `custom` | Custom | User-defined Python services |

## 2. `service_registry`

One row per running service instance. The `base_url` field determines where the frontend sends requests.

```sql
CREATE TABLE public.service_registry (
  service_id      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  service_type    TEXT NOT NULL
                    REFERENCES public.service_type_catalog(service_type),
  service_name    TEXT NOT NULL,
  base_url        TEXT NOT NULL,
  health_status   TEXT NOT NULL DEFAULT 'unknown'
                    CHECK (health_status IN ('online', 'offline', 'degraded', 'unknown')),
  last_heartbeat  TIMESTAMPTZ,
  config          JSONB NOT NULL DEFAULT '{}'::jsonb,
  enabled         BOOLEAN NOT NULL DEFAULT true,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),

  UNIQUE (service_type, service_name)
);
```

**Key columns:**

| Column | Type | Notes |
|---|---|---|
| `service_id` | `UUID` | Auto-generated PK |
| `service_type` | `TEXT` | FK to `service_type_catalog` |
| `service_name` | `TEXT` | Unique within type (e.g. `eyecite`, `load-runner`) |
| `base_url` | `TEXT` | Full base URL (e.g. `http://localhost:8000`) |
| `health_status` | `TEXT` | `online` / `offline` / `degraded` / `unknown` |
| `config` | `JSONB` | Service-level config (version, credentials ref, etc.) |
| `enabled` | `BOOLEAN` | Soft-disable toggle |

**Uniqueness:** `(service_type, service_name)` — one service per type+name combination.

## 3. `service_functions`

One row per callable function. Each function belongs to one service. The `entrypoint` is the path appended to the service's `base_url`.

```sql
CREATE TABLE public.service_functions (
  function_id       UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  service_id        UUID NOT NULL
                      REFERENCES public.service_registry(service_id) ON DELETE CASCADE,
  function_name     TEXT NOT NULL,
  function_type     TEXT NOT NULL
                      CHECK (function_type IN (
                        'source', 'destination', 'transform', 'parse', 'convert',
                        'export', 'test', 'utility', 'macro', 'custom',
                        'ingest', 'callback'
                      )),
  label             TEXT NOT NULL,
  description       TEXT,
  entrypoint        TEXT NOT NULL,
  http_method       TEXT NOT NULL DEFAULT 'POST'
                      CHECK (http_method IN ('GET', 'POST', 'PUT', 'DELETE')),
  parameter_schema  JSONB NOT NULL DEFAULT '[]'::jsonb
                      CHECK (jsonb_typeof(parameter_schema) = 'array'),
  result_schema     JSONB DEFAULT NULL,
  enabled           BOOLEAN NOT NULL DEFAULT true,
  tags              JSONB NOT NULL DEFAULT '[]'::jsonb
                      CHECK (jsonb_typeof(tags) = 'array'),
  created_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at        TIMESTAMPTZ NOT NULL DEFAULT now(),

  UNIQUE (service_id, function_name)
);
```

**Key columns:**

| Column | Type | Notes |
|---|---|---|
| `function_id` | `UUID` | Auto-generated PK |
| `service_id` | `UUID` | FK to `service_registry`, CASCADE delete |
| `function_name` | `TEXT` | Unique within service (e.g. `eyecite_clean`) |
| `function_type` | `TEXT` | 12 valid values — see CHECK constraint |
| `entrypoint` | `TEXT` | Path relative to `base_url` (e.g. `/eyecite/clean`) |
| `http_method` | `TEXT` | `GET` / `POST` / `PUT` / `DELETE` |
| `parameter_schema` | `JSONB` | Array of param definitions (see below) |
| `result_schema` | `JSONB` | Optional — expected result shape |
| `tags` | `JSONB` | Searchable string array |

**`parameter_schema` format:** Each element is an object:

```json
{
  "name": "text",
  "type": "string",
  "required": true,
  "default": null,
  "description": "Raw input text.",
  "values": null
}
```

| Field | Type | Required | Notes |
|---|---|---|---|
| `name` | `string` | yes | Parameter name |
| `type` | `string` | yes | `string`, `boolean`, `number`, `json`, `enum`, `secret` |
| `required` | `boolean` | no | Default `false` |
| `default` | `any` | no | Default value |
| `description` | `string` | no | Help text for UI |
| `values` | `string[]` | no | Valid choices (for `enum` type) |

**Indexes:**

```sql
CREATE INDEX service_functions_service_id_idx   ON public.service_functions (service_id);
CREATE INDEX service_functions_function_type_idx ON public.service_functions (function_type);
CREATE INDEX service_functions_tags_idx          ON public.service_functions USING GIN (tags);
```

## 4. `service_runs`

Execution history for any function. One row per invocation.

```sql
CREATE TABLE public.service_runs (
  run_id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  function_id     UUID NOT NULL
                    REFERENCES public.service_functions(function_id) ON DELETE CASCADE,
  service_id      UUID NOT NULL
                    REFERENCES public.service_registry(service_id) ON DELETE CASCADE,
  project_id      UUID
                    REFERENCES public.projects(project_id) ON DELETE SET NULL,
  status          TEXT NOT NULL DEFAULT 'pending'
                    CHECK (status IN ('pending', 'running', 'complete', 'failed', 'cancelled')),
  config_snapshot JSONB NOT NULL DEFAULT '{}'::jsonb,
  result          JSONB,
  error_message   TEXT,
  rows_affected   BIGINT,
  started_at      TIMESTAMPTZ,
  completed_at    TIMESTAMPTZ,
  duration_ms     DOUBLE PRECISION,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**Indexes:**

```sql
CREATE INDEX service_runs_function_id_idx ON public.service_runs (function_id);
CREATE INDEX service_runs_service_id_idx  ON public.service_runs (service_id);
CREATE INDEX service_runs_project_id_idx  ON public.service_runs (project_id);
CREATE INDEX service_runs_status_idx      ON public.service_runs (status);
CREATE INDEX service_runs_created_at_idx  ON public.service_runs (created_at DESC);
```

## 5. `service_run_events`

Append-only event/log stream per run. Intended for real-time UI subscriptions.

```sql
CREATE TABLE public.service_run_events (
  event_id   BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  run_id     UUID NOT NULL
               REFERENCES public.service_runs(run_id) ON DELETE CASCADE,
  ts         TIMESTAMPTZ NOT NULL DEFAULT now(),
  level      TEXT NOT NULL DEFAULT 'INFO'
               CHECK (level IN ('TRACE', 'DEBUG', 'INFO', 'WARN', 'ERROR')),
  message    TEXT NOT NULL DEFAULT '',
  raw_event  JSONB,
  source     TEXT
);

CREATE INDEX service_run_events_run_id_ts_idx
  ON public.service_run_events (run_id, ts ASC);
```

## 6. `service_run_artifacts`

Pointers to stored outputs (Supabase Storage).

```sql
CREATE TABLE public.service_run_artifacts (
  artifact_id    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  run_id         UUID NOT NULL
                   REFERENCES public.service_runs(run_id) ON DELETE CASCADE,
  name           TEXT NOT NULL,
  storage_bucket TEXT NOT NULL,
  storage_key    TEXT NOT NULL,
  sha256         TEXT,
  bytes          BIGINT,
  created_at     TIMESTAMPTZ NOT NULL DEFAULT now(),

  UNIQUE (run_id, name)
);

CREATE INDEX service_run_artifacts_run_id_idx
  ON public.service_run_artifacts (run_id);
```

## 7. `service_functions_view`

Convenience view joining functions + services + type labels. Only shows enabled rows.

```sql
CREATE OR REPLACE VIEW public.service_functions_view AS
SELECT
  sf.function_id,
  sf.function_name,
  sf.function_type,
  sf.label,
  sf.description,
  sf.entrypoint,
  sf.http_method,
  sf.parameter_schema,
  sf.result_schema,
  sf.enabled       AS function_enabled,
  sf.tags,
  sr.service_id,
  sr.service_type,
  sr.service_name,
  sr.base_url,
  sr.health_status,
  sr.enabled       AS service_enabled,
  stc.label        AS service_type_label
FROM public.service_functions sf
JOIN public.service_registry sr    ON sr.service_id = sf.service_id
JOIN public.service_type_catalog stc ON stc.service_type = sr.service_type
WHERE sf.enabled = true AND sr.enabled = true;
```

## RLS Policies

All tables use RLS. Summary:

| Table | `authenticated` | `service_role` |
|---|---|---|
| `service_type_catalog` | SELECT | ALL |
| `service_registry` | SELECT | ALL |
| `service_functions` | SELECT | ALL |
| `service_runs` | SELECT + INSERT (own projects) | ALL |
| `service_run_events` | SELECT (own projects) | ALL |
| `service_run_artifacts` | SELECT (own projects) | ALL |

**"Own projects"** = `project_id IN (SELECT project_id FROM projects WHERE owner_id = auth.uid())`

Admin mutations (create/update/delete services and functions) go through the FastAPI `service_role` client, bypassing RLS.

## Realtime

Both `service_registry` and `service_functions` are added to the `supabase_realtime` publication (migration 056). The frontend subscribes to `postgres_changes` events on these tables for live updates.

---

## FastAPI Admin Routes

All admin CRUD is served by the pipeline-worker FastAPI service at `http://localhost:8000`:

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/admin/services` | List all services, functions, and service types |
| `POST` | `/admin/services/service` | Create a service |
| `PATCH` | `/admin/services/service/{service_id}` | Update a service |
| `DELETE` | `/admin/services/service/{service_id}` | Delete a service (cascades to functions) |
| `POST` | `/admin/services/function` | Create a function |
| `PATCH` | `/admin/services/function/{function_id}` | Update a function |
| `DELETE` | `/admin/services/function/{function_id}` | Delete a function |
| `POST` | `/admin/services/import` | Bulk import (services + functions + Kestra plugins) |

All routes require `Authorization: Bearer <supabase_jwt>` from a superuser (email in `SUPERUSER_EMAIL_ALLOWLIST`).
