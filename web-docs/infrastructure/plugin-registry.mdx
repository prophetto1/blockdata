## Overview

Blockdata's plugin registry maps third-party integration capabilities to executable service functions. Each catalog item (originating from Kestra's plugin ecosystem) is being mapped to a `service_registry` entry and a `service_functions` row, creating the link between the catalog definition and the FastAPI execution layer.

**Total catalog items:** 945
**Total plugin groups:** 166
**Mapped to service_functions:** 2 / 945
**Custom blockdata additions:** TBD (user-defined, target 250+)
**Combined target:** ~1,200+

## Database Schema

The mapping chain:

```
integration_catalog_items.task_class
  ‚Üí mapped_service_id  ‚Üí service_registry (service_name, service_type, base_url)
  ‚Üí mapped_function_id ‚Üí service_functions (function_name, parameter_schema, entrypoint)
```

**Execution flow:**

```
UI ‚Üí POST /execute { plugin_type, params }
  ‚Üí FastAPI resolves plugin handler
  ‚Üí handler calls third-party API with params
  ‚Üí returns PluginOutput { data, state, logs }
  ‚Üí service_runs row recorded
```

## Service Types

| Type | Label | Scope |
|------|-------|-------|
| `notification` | Notifications | Slack, Discord, email, Telegram, SMS |
| `cloud` | Cloud Providers | AWS, Azure, GCP |
| `database` | Databases | Postgres, MySQL, Snowflake, MongoDB, Redis, etc. |
| `messaging` | Message Queues | Kafka, AMQP, NATS, MQTT, Pulsar |
| `saas` | SaaS Integrations | HubSpot, Stripe, Shopify, Airtable, Notion, etc. |
| `ai` | AI / LLM | Anthropic, OpenAI, Gemini, Ollama, etc. |
| `devops` | DevOps / Infra | Docker, Kubernetes, Git, Terraform, etc. |
| `script` | Script Execution | Python, Shell, Node, Go, R, etc. |
| `transform` | Data Transform | dbt, serialization, compression |
| `load` | Data Load | dlt, Airbyte, Fivetran |
| `filesystem` | File Systems | FTP, SFTP, SCP, S3-compatible, Azure Blob |
| `core` | Core / Flow Control | Log, Sleep, If, Switch, ForEach, Parallel |
| `custom` | Custom | User-defined Python services |

<Note>
Types `dbt`, `dlt`, `docling`, `edge`, `conversion`, `custom` already exist in `service_type_catalog`. New types above will be added via migration as groups are mapped.
</Note>
