## Page Layout Standard

Rules for admin page frames inside the settings shell.

| Rule | Detail |
|---|---|
| Header title only | Each admin page frame shows the page title in the header bar. No group label pills, no sub-tab buttons, no separator below the header. |
| Sidebar owns navigation | The settings second rail (left nav) is the sole navigation surface. Peer pages within a group are listed there — never duplicated inside the content area. |
| No redundant context badges | Group names (Runtime, Operations, Design) appear only in the sidebar group headers, not repeated as pills or breadcrumbs inside the page. |
| Full-bleed content | After the header, the content area fills all remaining vertical space with standard padding (`p-3 md:p-4`). No intermediate chrome between header and content. |

<Accordion title="Frame Anatomy">

```
┌─────────────────────────────────────────────┐
│  Page Title                  [Action Button] │  ← border-b only
├─────────────────────────────────────────────┤
│                                             │
│  Content area                               │
│  (fills remaining height)                   │
│  padding: p-3 md:p-4                        │
│                                             │
└─────────────────────────────────────────────┘
```

</Accordion>

## Spacing

Tailwind's default spacing scale. Base unit: `0.25rem` (4px).

| Class | rem | Use |
|---|---|---|
| `gap-1` / `p-1` | 0.25 | Tight internal padding |
| `gap-2` / `p-2` | 0.5 | Compact element spacing |
| `gap-3` / `p-3` | 0.75 | Inline sibling spacing |
| `gap-4` / `p-4` | 1.0 | Section padding |
| `gap-5` / `p-5` | 1.25 | Generous padding |
| `gap-6` / `p-6` | 1.5 | Vertical section rhythm |

<Check>
- Inline spacing between siblings: `gap-3` or `gap-4`
- Section padding: `p-4` to `p-5`
- Page-level gutters: `p-4` mobile, `p-5` desktop
- Vertical rhythm between sections: `gap-6`
- Never use raw pixel values
</Check>

| `md:` | ≥ 48em (768px) |
| `lg:` | ≥ 64em (1024px) |

## Borders & Radius

**Border width:** `border` (1px) always. No 2px borders.

**Border color:** `border-border` (resolves to `--border` token).

| Tailwind class | Value | Use |
|---|---|---|
| `rounded-sm` | `calc(var(--radius) - 4px)` | Badges, chips, small pills |
| `rounded-md` | `calc(var(--radius) - 2px)` | Buttons, inputs, selects |
| `rounded-lg` | `var(--radius)` (0.625rem) | Cards, modals, panels |
| `rounded-xl` | `calc(var(--radius) + 4px)` | Large containers |
| `rounded-full` | 9999px | Avatars, dots |

## Dark Mode

Dark is the default. Light activates via `data-theme="light"` on `<html>`.

<Accordion title="Rules">
  Use Tailwind semantic color utilities (`bg-background`, `text-foreground`, `border-border`). They resolve automatically per mode.

  For one-off mode differences, use Tailwind's `dark:` variant.

  Never detect mode in JS. Never toggle classes manually for dark/light.

  Never use `light-dark()` CSS function (Mantine pattern — banned).
</Accordion>

## Component Styling Rules

All styles are Tailwind utility classes directly on JSX elements. No exceptions.

<Accordion title="The Rules" defaultOpen>
  No `.css` files per component (except legacy shell components being migrated).

  No global CSS class names (`.my-component-header`).

  No CSS-in-JS or styled-components.

  No `style` props except for truly dynamic values.

  Use `cn()` (clsx + twMerge) for conditional class composition.

  Ark UI uses `data-selected`, `data-disabled`, `data-focus` — NOT `data-state="active"`.
</Accordion>
