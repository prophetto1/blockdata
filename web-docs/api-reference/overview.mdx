## Base URL

The pipeline worker is an internal FastAPI service. It is not exposed to the public internet — all external access goes through the Supabase edge function layer.

| Environment | Base URL |
|---|---|
| Local development | `http://localhost:8000` |
| Docker network | `http://pipeline-worker:8000` |
| LAN (Ubuntu box) | `http://<host-ip>:8000` |

## Authentication

The pipeline worker has no authentication on the inbound side. It trusts all requests from the private network.

| Layer | Auth | Enforced by |
|---|---|---|
| Browser → Supabase edge function | Supabase JWT | Edge function |
| Edge function → pipeline worker | None | Network isolation |
| Pipeline worker → external APIs | Per-task (API keys, OAuth) | Plugin code |

When the task dispatcher edge function ships, it will add JWT verification on the public side and proxy to the worker. The worker itself stays unauthenticated.

## Request format

Every task goes through one endpoint:

```
POST /execute
Content-Type: application/json
```

```json
{
  "task_type": "blockdata.eyecite.extract",
  "params": {
    "text": "Roe v. Wade, 410 U.S. 113 (1973)"
  },
  "execution_id": "",
  "task_run_id": "",
  "variables": {}
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `task_type` | string | yes | Plugin dispatch key. Determines which handler runs. |
| `params` | object | no | Parameters passed to the plugin. Schema varies per task type. |
| `execution_id` | string | no | Caller-assigned execution ID for tracing. |
| `task_run_id` | string | no | Caller-assigned task run ID for tracing. |
| `variables` | object | no | Template variables available as `{{ path.to.value }}` inside params. |

### Task type format

Task types follow a dotted namespace:

| Provider | Pattern | Example |
|---|---|---|
| Core | `io.kestra.plugin.core.*` | `io.kestra.plugin.core.log.Log` |
| HTTP | `io.kestra.plugin.core.http.*` | `io.kestra.plugin.core.http.Request` |
| Scripts | `io.kestra.plugin.scripts.*` | `io.kestra.plugin.scripts.python.Script` |
| eyecite | `blockdata.eyecite.*` | `blockdata.eyecite.extract` |

Most plugins also accept legacy aliases (e.g. `io.kestra.core.tasks.log.Log`). See each provider page for the full list.

### Template expressions

String parameters support `{{ expression }}` syntax. The worker resolves dotted paths against the `variables` object before passing params to the plugin.

```json
{
  "task_type": "io.kestra.plugin.core.log.Log",
  "params": {
    "message": "Processing {{ outputs.step1.count }} items"
  },
  "variables": {
    "outputs": {
      "step1": { "count": 42 }
    }
  }
}
```

Renders `message` as `"Processing 42 items"`.

## Response format

Every response wraps the plugin output in a standard envelope:

```json
{
  "task_type": "blockdata.eyecite.extract",
  "output": {
    "data": { ... },
    "state": "SUCCESS",
    "logs": ["Found 3 citations"]
  }
}
```

### Output fields

| Field | Type | Description |
|---|---|---|
| `data` | object | Plugin-specific output. Shape varies per task type. |
| `state` | string | `SUCCESS`, `FAILED`, `WARNING`, or `PAUSED`. |
| `logs` | string[] | Human-readable log messages from the plugin. |

### State values

| State | Meaning |
|---|---|
| `SUCCESS` | Task completed normally. |
| `FAILED` | Task encountered an error. `data` may contain partial results. `logs` contains the error message. |
| `WARNING` | Task completed with caveats (e.g. HTTP 4xx response). |
| `PAUSED` | Task requests the orchestrator to pause execution (Pause plugin only). |

## Error handling

**Plugin errors** return HTTP 200 with `state: "FAILED"`:

```json
{
  "task_type": "blockdata.eyecite.extract",
  "output": {
    "data": {},
    "state": "FAILED",
    "logs": ["text is required"]
  }
}
```

**Unknown task type** returns HTTP 404:

```json
{
  "detail": "No handler for task type: nonexistent.type"
}
```

**Unhandled exceptions** return HTTP 200 with `state: "FAILED"` and the exception message in `logs`.

## Discovery endpoints

### Health check

```
GET /health
```

```json
{
  "status": "ok",
  "plugins": 18
}
```

### List all plugins

```
GET /plugins
```

Returns every registered plugin class with its task types and parameter schema:

```json
[
  {
    "class": "LogPlugin",
    "task_types": [
      "io.kestra.plugin.core.log.Log",
      "io.kestra.core.tasks.log.Log"
    ],
    "parameter_schema": [
      {
        "name": "message",
        "type": "string",
        "required": true,
        "description": "Message to log. Supports {{ expressions }}."
      },
      {
        "name": "level",
        "type": "string",
        "required": false,
        "default": "INFO",
        "values": ["TRACE", "DEBUG", "INFO", "WARN", "ERROR"]
      }
    ]
  }
]
```
