## Purpose

This document consolidates the question types currently used (or planned) in Legal-10 into a single question bank. Each question type is expressed in atomic form so it can be selected in a UI, compiled into a runspec, and deterministically expanded into EU packages (prompts, ground truth artifacts, and scorers).

This QB also tracks whether each template is **implementation-ready** (a "Fully Developed Question") per:

- `internal/specs/steps/_defining_fully_developed_question.md`

Each template below includes a short **Development status** line: `Fully developed`, `Partially developed`, or `Deferred`.

---

## Atomic question unit contract (high-level)

Each question unit should be representable as:

* a stable question type id (e.g., KA-SC / CANARY / TRANSITIVE\_AUTHORITY)
* a template id (for templated questions; e.g., 1.1) or a fixed step id (for single-instance steps; e.g., 4.1)
* candidate-facing payload (admitted text + provided options)
* expected model output schema (closed-form where possible)
* runner-only ground truth payload (or references)
* deterministic scoring function (or judge-scored rubric reference)
* coverage predicate (when the question type is eligible for this anchor)
* chain-position constraints (if any)

---

## Chain position constraints (summary)

Some steps have hard constraints even in a runspec-agnostic system:

* Citation Integrity (4.x) must run after the evaluated model completes its final drafted output(s) (end-of-chain verification).
* IRAC without RP (6.x) must occur immediately before IRAC with RP (7.x) (paired drafting).
* IRAC with RP (7.x) is currently designed as the last candidate-facing drafting step.
* Known Authority (1.x) is typically first, but only requires the anchor citation universe; it can be fixed to Step 1 if desired.
* Canary (3.x) can be standalone and/or RNG-randomized in position depending on what you want to test (pre-context vs post-context discipline).

# 1. KA-SC (Known Authority - SCOTUS Universe)

Development status: Partially developed as a family. Most KA-SC templates are fully developed; hybrid backbone scoring is still policy/data-dependent (see template-level status).

Known Authority is answer-sheet driven. The evaluated model chooses only from citations extracted from the anchor. KA-SC uses the SCOTUS universe with Fowler authority scores.

| Type | Deterministic |

# Appendix E. Design Intent and Legacy Mapping

This appendix documents the original design intent behind steps 8-11 and clarifies how they relate to the KA family. It also provides a Chain2 atomic unit contract reference and a legacy-to-QB mapping table.

## E.1 Legacy Step Intent (S2-S5)

### S2: Unknown Authority (reverse-citator retrieval)

**Intent:** Given a target SCOTUS case (the precedent), the model should name SCOTUS cases that cite it. Scoring uses MRR against a known true citing case embedded in the dataset instance.

**What makes it non-lazy:** It tests recall/research-like retrieval under constraint (no browsing) and rewards higher-ranked correct mentions.

### S3: Authority Status (dataset-annotation reporting, not legal judgment)

**Intent:** Force the model to report (not opine) what two sources say:
- Curated overrule annotation ("Dahl" in the code)
- Shepard's treatment for a specific citing->cited pair

Then emit a deterministic "consistency\_flag".

**Important:** This is explicitly NOT a good-law determination; it is metadata reporting with a mandatory caveat.

### S4: Fact Extraction (structured case outcome)

**Intent:** From an opinion text, extract outcome fields (disposition and party\_winning) and a short holding summary, plus evidence quotes. Scoring is deterministic against SCDB metadata for disposition and party\_winning (holding summary/quotes are not scored in the current code).

### S5: Distinguish (edge-level treatment classification)

**Intent:** Given a cited precedent and a citing case (optionally with citing-opinion excerpt), classify whether the citing case agrees or distinguishes and optionally name a normalized treatment label. Scoring is deterministic against ground truth derived from the Shepard's edge (edge.agree + normalize\_shepards\_to\_treatment).

**Important:** S5 is NOT KA as defined in the KA Question Bank. S5 is edge-level treatment classification. KA is anchor-level selection among citations that appear in the anchor.

---

## E.2 What is NOT KA (Separate Atomic Families)

High overlap with the KA Question Bank:
- Any "pick/rank the most authoritative cited case" question among anchor-cited options
- Any "bucket cited cases into supporting/limiting/neutral" among anchor-cited options
- Any frequency/prominence question (counts, first\_offset) among anchor-cited options

These should remain in the KA family (1.x and 2.x):
- KA-SC templates: SC-A\*, SC-P\*, SC-F\*, SC-H\*
- KA-CAP templates: CAP-A\*, CAP-F\*, optional CAP-P\*

What is NOT KA (separate atomic families in 8.x-11.x):
- **S2 / Unknown Authority (8.x):** operates on "cases that cite the precedent", not "citations within the anchor"
- **S3 / Validate Authority (9.x):** operates on (curated overrule + Shepard's pair). It is metadata reporting.
- **S5 / Distinguish (11.x):** operates on a specific citing->cited pair. It is not an anchor-candidate selection task.
- **S4 / Fact Extraction (10.x):** case outcome extraction (anchor-agnostic).

This separation prevents conceptual drift (e.g., mixing edge-level treatment classification into KA).

## E.3 Chain2 Atomic Unit Contract (Reference)

A minimal stable core for any atomic question unit:

| Field | Description |
|-------|-------------|
| **atom\_id** | stable string (e.g., "ka\_sc\_authority\_pair", "rc\_list\_citing\_cases\_v1") |
| **family** | e.g., "KA", "CitatorEdge", "FactExtraction", "ReverseCitator" |
| **coverage()** | deterministic boolean based on instance + available artifacts |
| **requires** | list of upstream atoms (optional) |
| **build\_prompt(ctx)** | returns prompt text + model-visible payload |
| **response\_schema** | JSON schema (closed) |
| **ground\_truth(ctx)** | deterministic, uses only frozen artifacts (never model output) |
| **score(parsed, gt)** | deterministic scoring |
| **instantiate(anchor\_caseId)** | deterministic question generator (select options, shuffle with seed) |
