# Kestra CT

Control workspace for the Kestra integration effort.

Purpose:
- keep the program-level plan in one place
- keep worker instructions and templates stable
- track page packets, capture docs, implementation docs, and verification docs
- prevent page workers from inventing different methods

Initial scope:
- assessment refresh
- preparation-first implementation plan
- worker system and templates
- page registry and verification matrix
- first-page packet for the golden path

Layout:
- `plans/` implementation plans and readiness plans
- `assessments/` plan reviews and go/no-go gates
- `onboarding/` worker-facing instructions, onboarding docs, and templates
- future `pages/` page packets and page-specific docs
- `decisions.md` and `page-registry.yaml` as root control files
