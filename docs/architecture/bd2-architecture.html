3. Crew Execution Pipeline

How a user defines a crew, runs it, and monitors results — entirely missing today.

%%{init: {'theme': 'dark', 'themeVariables': {'fontSize': '13px'}}}%%
sequenceDiagram
    participant U as User · Browser
    participant FE as React Frontend
    participant API as FastAPI Backend
    participant Q as Job Queue · Celery
    participant W as Worker Process
    participant CR as crewAI Engine
    participant LLM as LLM Provider
    participant DB as Supabase · Postgres
    participant VEC as pgvector · Embeddings

    U->>FE: Define agents + tasks
    FE->>DB: Save crew_definitions

    U->>FE: Click "Run Crew"
    FE->>API: POST /crews/{id}/kickoff
    API->>DB: Insert execution_record (PENDING)
    API->>Q: Enqueue job
    API-->>FE: Return execution_id

    Q->>W: Job picked up
    W->>DB: Load crew definitions
    W->>DB: Update status → RUNNING
    W->>CR: crew.kickoff(inputs)

    loop Agent execution
        CR->>VEC: Query knowledge (blocks)
        VEC-->>CR: Relevant chunks
        CR->>LLM: Prompt + knowledge context
        LLM-->>CR: Response
        CR->>DB: Log event (execution_events)
    end

    CR-->>W: Final result
    W->>DB: Update status → SUCCESS + result
    DB-->>FE: Realtime subscription fires
    FE-->>U: Dashboard shows result

5. Technology Stack

Presentation

  • React + TypeScript
  • Workbench (multi-pane)
  • SchemaLayout (3-column)
  • FlowsList (list/table)
  • SettingsAdmin (config panels)

API Layer

  • Supabase Edge Functions (TS/Deno)
  • FastAPI (Python) — expand
  • Supabase JS Client (realtime + auth)

Processing

  • Docling (document AI)
  • crewAI (agent framework) — integrate
  • Celery + Redis (job queue) — add
  • Embedding providers — add

Data

  • PostgreSQL via Supabase
  • pgvector (vector search) — add
  • Supabase Storage (files)
  • ArangoDB (graph, if needed)
