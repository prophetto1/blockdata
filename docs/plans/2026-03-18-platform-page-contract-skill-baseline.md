# Platform Page Contract Skill Baseline

This note captures the gap the third skill needs to close.

## Pressure Scenario

One page is the whole scope. The user wants a deterministic package that captures that page's shell, day and night tokens when available, responsive shell behavior, and visible component inventory without drifting into multi-route analysis or redesign advice.

## Baseline Risks Without The Skill

- Expand one page into a pseudo-platform crawl.
- Produce different output files from the same page inputs.
- Omit one required theme when the page supports day and night.
- Skip responsive shell differences across desktop, tablet, and mobile.
- Treat visible components as optional instead of part of the page contract.

## Success Criteria For The Skill

- Stay scoped to one page only.
- Emit one package shape every run.
- Require both themes when the page supports both.
- Include shell, tokens, typography, spacing, breakpoints, and visible/openable component inventory.
- Allow user help only when theme switching cannot be completed deterministically.
